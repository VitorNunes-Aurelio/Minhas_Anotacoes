# Git
O git é uma pasta(ou repositorio, ja que significa a mesma coisa) para armazenar os seus arquivos de forma local. Porém, também pode ser realizado comit no repositorio para salvar as alterações no arquivo 

# Github 
O github é um servidor para salvar os arquivos e as alterações, funcionado como uma forma (quase) definitiva do projeto. E as alterações do git para o github são chamados de push, e o inverso é pull.
